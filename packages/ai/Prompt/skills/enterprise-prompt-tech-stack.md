# AI提示词企业级技术栈规范

严格遵循企业技术栈统一原则，与现有全栈开发体系无缝对齐，禁止使用未纳入首选清单的工具/框架，特殊替代方案需AI技术委员会与安全合规委员会双审批。

## 首选技术栈对照表

| 开发方向             | 唯一首选技术栈                     | 禁止/不推荐替代方案              | 核心说明                                                                                              |
| -------------------- | ---------------------------------- | -------------------------------- | ----------------------------------------------------------------------------------------------------- |
| 提示词核心框架       | LangChain.js ChatPromptTemplate    | 硬编码字符串模板、自定义模板封装 | 与企业现有LangChain/LangGraph技术栈100%兼容，原生支持TS类型校验、变量管理、结构化输出                 |
| 输入输出类型校验     | Zod                                | Yup、Joi、无校验方案             | 与企业全栈规范完全对齐，一套Schema同时覆盖TS类型推导、LLM输出格式约束、运行时校验，实现全链路类型安全 |
| 提示词版本管理       | Git + 语义化版本                   | 在线文档、本地文件、无版本管控   | 与企业代码管理规范对齐，所有提示词资产纳入Git仓库，变更可追溯、可回滚                                 |
| 提示词测试与可观测性 | LangSmith                          | 自定义测试脚本、无测试方案       | 官方原生适配LangChain生态，支持提示词调试、效果评估、全链路追踪、生产环境监控                         |
| 提示词安全防护       | Lakera Guard + Zod输出校验         | 无防护方案、自定义过滤规则       | 企业级提示词注入、越狱、敏感信息防护，配合Zod实现输入+输出双端安全管控                                |
| 结构化输出           | LangChain.js 结构化输出 + Zod      | 纯文本格式约束、无结构化方案     | 强制LLM输出符合Zod Schema的结构化内容，杜绝格式异常导致的业务故障                                     |
| 提示词编排           | LangGraph.js                       | 线性Chain、自定义流程封装        | 与企业智能体开发规范对齐，实现提示词与工具调用、业务节点的可控编排                                    |
| 环境与配置管理       | Next.js 环境变量 + Vercel 环境隔离 | 硬编码密钥、环境混用             | 与企业部署规范完全对齐，实现开发/预发/生产三环境隔离，敏感信息零泄露                                  |
| 日志与审计           | Pino + Sentry                      | console.log、无日志方案          | 与企业日志规范对齐，全量提示词调用、变更、异常事件可审计、可追溯、可告警                              |

## 使用规范

1. 新项目必须严格按照首选技术栈选型，禁止私自引入未授权技术
2. 存量项目逐步迁移至统一技术栈，禁止新增代码使用非首选技术
3. 所有技术选型必须与TypeScript+Next.js全栈开发规范、以太坊开发规范、LangChain/LangGraph技术栈完全兼容对齐
