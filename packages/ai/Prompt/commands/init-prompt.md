# 初始化企业级提示词

根据《AI提示词开发企业级规范》创建标准化的提示词工程结构，包含所有强制要求的模块与配置。

## 执行步骤
1. 确认提示词的业务场景、核心目标与成功指标
2. 创建独立文件，命名格式为 `[提示词名称].prompt.ts`，存放于对应业务模块目录
3. 按照固定顺序添加8个核心模块，无对应内容标注「无特殊要求」，不得省略：
   - 角色定位：精准定义身份、领域与能力边界
   - 核心目标：1-2句话明确唯一核心任务
   - 执行规则：分点列出可落地的执行步骤
   - 输入规范：明确所有输入变量的含义、类型、约束，对齐Zod Schema
   - 输出规范：明确输出格式结构，对齐Zod Schema
   - 约束与禁忌：列出绝对禁止的行为与合规要求
   - 异常处理规则：明确所有异常场景的处理方式
   - 参考示例（Few-Shot）：复杂场景提供2-3组覆盖不同情况的示例
4. 创建配套的Zod Schema文件，分别定义输入与输出类型
5. 确认未硬编码任何敏感信息，所有敏感配置通过环境变量管理
6. 确认符合所有编写强制红线要求

## 检查要点
- ✅ 结构完整，8个核心模块无省略
- ✅ 输入输出与Zod Schema完全对齐，可推导TS类型
- ✅ 无模糊描述，所有规则可落地可校验
- ✅ 无泛化无边界任务，明确能力边界
- ✅ 无硬编码敏感信息
