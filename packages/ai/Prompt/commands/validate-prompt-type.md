# 校验提示词类型安全

按照企业类型安全强制规范，检查提示词输入输出的类型定义与校验逻辑。

## 执行步骤
1. 检查输入变量：确认所有输入都定义了Zod校验Schema与对应TS类型
2. 确认输入流程：用户输入先经过Zod校验，校验不通过直接拦截，不会传入模型
3. 检查输出定义：确认输出已定义Zod Schema，使用LangChain结构化输出能力约束格式
4. 确认输出流程：模型输出后必须经过Zod二次校验，校验不通过触发异常处理，不会流入下游
5. 检查类型复用：Schema定义与业务代码复用同一套类型，无重复定义导致不一致
6. 工具调用场景额外检查：工具入参Schema与Zod定义完全一致，无未授权工具调用可能

## 检查要点
- ✅ 所有输入输出都有对应的Zod Schema
- ✅ 输入输出都执行了Zod校验，无漏校验场景
- ✅ 类型与业务代码复用，无类型断层
- ✅ 工具调用入参符合Schema约束
