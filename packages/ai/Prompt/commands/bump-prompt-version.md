# 升级提示词版本

遵循语义化版本规范与企业变更管理要求，完成提示词版本升级。

## 执行步骤
1. 根据变更内容确定版本号升级类型：
   - 主版本号：不兼容的架构/核心规则/业务目标变更，升级第一位 `x.0.0`
   - 次版本号：新增功能、规则优化、示例补充，向下兼容，升级第二位 `x.y.0`
   - 补丁版本号：bug修复、文案优化、约束补充，向下兼容，升级第三位 `x.y.z`
2. 按照Conventional Commits规范编写Git提交信息，明确变更内容与变更原因
3. 确认所有测试用例执行通过，生成完整测试报告
4. 发起Code Review，至少获得1名AI提示词工程师与1名业务负责人审批
5. 合并代码后，打对应语义化Git Tag，关联测试报告与上线审批记录
6. 确认变更记录可追溯，支持快速回滚

## 检查要点
- ✅ 版本号升级符合语义化规则
- ✅ 提交信息符合Conventional Commits规范
- ✅ 所有测试用例执行通过
- ✅ Code Review完成审批
- ✅ Git Tag已创建，变更归档完整
- ✅ 支持快速回滚至上一版本
