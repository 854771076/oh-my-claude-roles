# 生成提示词测试用例

按照企业测试验收规范，生成覆盖全场景的标准化测试用例，适配Vitest+LangSmith测试框架。

## 执行步骤
1. 设计单元测试用例，必须覆盖四类场景：
   - 正常业务场景：覆盖核心业务流程
   - 边界输入场景：覆盖输入最大长度、极端值等边界情况
   - 异常输入场景：覆盖非法输入、缺失参数等异常
   - 攻击测试场景：覆盖常见提示词注入、越狱尝试
2. 按照验收标准设计用例检查点：
   - 格式合规：所有输出必须符合Zod Schema
   - 业务准确：核心场景必须符合业务规则
   - 异常处理：所有异常输入按预设规则处理
3. 设计集成测试用例：验证提示词在完整业务链路中的表现，覆盖上下游交互场景
4. 补充安全测试用例：覆盖注入、越狱、敏感信息泄露、违规生成等场景
5. 设计性能测试检查点：明确响应延迟、token消耗的达标要求
6. 将测试用例固化，适配Vitest+LangSmith执行，纳入CI/CD流程

## 检查要点
- ✅ 单元测试覆盖所有正常/边界/异常/攻击场景
- ✅ 每个用例有明确的可校验验收标准
- ✅ 集成测试覆盖完整业务链路
- ✅ 安全测试覆盖所有核心风险场景
- ✅ 适配企业统一测试框架，可自动化执行
