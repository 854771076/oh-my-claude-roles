## 规范要求

你必须严格按照 Claude Code Rules 官方规范生成规则文件。核心要求：

### 1. 文件格式
- 输出 **标准 Markdown 文件**（`.md` 后缀），UTF-8 编码
- 可选在文件顶部添加 YAML Frontmatter（用 `---` 包裹）定义路径匹配和元数据
- 完全兼容标准 Markdown 语法，支持标题、列表、代码块等

### 2. 可选 YAML Frontmatter 示例
```
---
# 路径匹配：限定规则仅对匹配的文件生效，!开头表示排除
paths:
  - "src/api/**/*.ts"
  - "!src/api/**/*.test.ts"
# 规则基础信息
name: API 开发规范
description: 项目 API 层的编码与安全标准
---
```

### 3. 内容结构规范
使用结构化标题组织内容：
- `#` 一级标题：规则核心主题（如 "Python 后端开发规范"）
- `##` 二级标题：规则分类（如 "核心编码规则"、"安全要求"、"测试规范"）
- `###` 三级标题：细分规则模块（如 "命名规范"、"类型注解要求"）

使用统一的约束级别表述：
- **must** / **必须**：强约束，强制执行的硬性要求
- **forbid** / **禁止**：强约束，绝对不允许的行为
- **recommend** / **建议**：弱约束，最佳实践推荐

使用代码块展示**正确示例**和**错误示例**，让规则更清晰。

### 4. 优先级规则
Claude Code 遵循"就近优先"原则：
- 越接近目标文件、生效范围越窄的规则，优先级越高
- 子目录规则 > 项目根规则 > 全局规则
- 具体明确的规则优先级高于模糊笼统的表述

## 提取要求

1. 提取原始文档中的编码规范、禁止项、强制要求、最佳实践
2. 按照上述结构化格式重新组织
3. 保持规则具体、明确、可执行
4. 只提取原始文档中明确提到的内容，不添加额外规则
5. 合理拆分分类，使用正确的标题层级

## 输出格式（严格遵守）

每个规则文件单独输出，使用以下格式：

## 文件名: 文件名.md

<完整文件内容，包含可选 frontmatter 和 Markdown 内容>

重点强调：
- `## 文件名:` 后面**必须只放文件名**，然后立即换行
- 文件名只能包含字母、数字、连字符 `-`、下划线 `_` 和点 `.`
- 不要在文件名那一行放任何其他内容
- 每个文件必须单独开头 `## 文件名:`
- 输出必须是标准 Markdown，不是 YAML！这是最关键的要求
