## Claude 合规要求

1. **文件结构**：每个技能单独一个 Markdown 文件，必须包含 YAML frontmatter
2. **Frontmatter 必填字段**：必须包含以下字段：
   - `name`: 技能名称（英文小写连字符）
   - `description`: 技能功能描述（中文）
   - `triggers`: 触发模式数组，匹配用户输入自动触发
3. **文档结构**：技能文档必须包含完整结构：
   - 标题：技能名称
   - 概述：技能解决什么问题
   - 核心功能：列出主要功能点
   - 使用方法：说明什么时候使用、怎么使用
   - 输入输出示例：提供实际使用示例
   - 注意事项：特殊说明和限制
4. **安全要求**：技能内容不得包含硬编码的敏感信息（如 API 密钥、密码、访问令牌等）
5. **语言一致性**：技能文档必须使用中文编写，与角色文档语言保持一致，表达清晰准确
6. **创建技能强制规范**：**必须**在技能文档的末尾添加以下规范说明块：

```markdown
---

## Claude 开发规范

创建新技能必须使用 `skill-create` / `skill-creator` 技能，遵循技能创建流程，不得手动创建。

```

注意：上面的代码块使用四个反引号在说明中，你输出时正常使用三个反引号。
7. **触发机制**：`triggers` 是匹配用户输入的模式数组，当用户输入包含任一模式时自动触发该技能
